DEMO 01

Running Your First Local LLM — LM Studio

What it is — LM Studio is a free desktop app (Mac / Windows / Linux) that downloads open-weight LLMs and runs them entirely on your own machine. No API key, no internet dependency at inference time, no per-token cost. The UI feels like ChatGPT, but nothing leaves your computer.

It supports pulling down several open-weight families:

Llama (Meta) · DeepSeek (reasoning) · Qwen · + others

Workflow

1 · Install app → 2 · Download weights (~5GB) → 3 · Load model → 4 · Chat

Why this demo matters (before touching any paid API)

It's deliberately the simplest possible entry point — and it quietly teaches several things at once:

  • Demystifies "the model" — you hold the weights as a file on disk. An LLM stops being a "magic cloud service" and becomes a program you download and run — a mental model you'll need later for cost, latency, privacy, and deployment decisions.
  • Zero cost, zero cloud dependency — a sandbox for experimentation and privacy-sensitive use cases (e.g. internal docs) before wiring up any paid API key.

Capabilities previewed (click to expand)

Generation & rewriting
Drafts a free-form email, then rewrites it in a different format/tone (email → poem) — shows controllable style transfer.
Explanation at variable complexity
Simplifies a technical topic (EV mechanics) for a target audience — a direct preview of prompt-engineering work later in the course.
Code generation
Writes a working ML script (e.g. a neural net to classify traffic signs) — foreshadows the coding work throughout the bootcamp.
Structured reasoning
Solves a physics problem by writing the governing equation and substituting values — an early look at chain-of-thought style behavior.
Document Q&A on an uploaded file — the big one
This is a live, working RAG (Retrieval-Augmented Generation) pipeline. LM Studio chunks the uploaded document, retrieves the relevant chunks for your question, and answers with a citation back to the source — demoed on a Tesla 10-K financial report. Months before you build a RAG pipeline from scratch in LangChain, you're watching one work end-to-end.
Architect's takeaway — This is the "hello world" of the whole bootcamp. In ~10 minutes it walks the full stack — model weights → local inference runtime → chat interface → basic retrieval-augmented Q&A — end to end, with zero infrastructure. Everything from Day 1 onward (APIs, prompting, RAG with LangChain/ChromaDB, agents) is a more powerful, more controllable version of what you just clicked through here.

RECALL

Active Recall Check

Click a card to reveal the answer
What made the LM Studio document-upload demo a live example of, and what steps happen behind the scenes??
A RAG (Retrieval-Augmented Generation) pipeline: the document is chunked, relevant chunks are retrieved for the question, and the model answers with a citation back to the source.
Why run an LLM locally via LM Studio before ever touching a paid API??
Zero cost and zero cloud dependency — a private sandbox for experimentation, and it reframes an LLM as "a program you run" rather than a cloud black box, which matters for later cost/latency/privacy/deployment decisions.
Which technique is introduced on Day 9 to fine-tune a model efficiently without retraining all its weights??
LoRA (Low-Rank Adaptation) — trains small adapter weights on top of the base model instead of the full parameter set.
Name the four agent/automation frameworks covered in Part C, and what distinguishes each.?
AutoGen (multi-agent + human-in-the-loop proxy), LangGraph (stateful graph workflows with conditional edges), CrewAI (role-based autonomous crews that execute code), n8n (no-code trigger-based automation).
What does MCP (Day 14) let an AI agent do??
Discover and call tools exposed by a server via a standardized protocol, using a "manifest" that lists the tools available.
